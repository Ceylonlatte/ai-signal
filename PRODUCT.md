# Product

## Register

product

## Users

单一用户——工具的作者本人，一个"挠自己痒"的自用工具（PRD 中标记为 Builder 模式）。

- **场景**：每天的固定动作，原本要手动刷 RSS / HackerNews / Reddit / Twitter 四个平台，既分散又耗时，刷过即忘。
- **要完成的事**：
  - 看到"今天哪些内容值得我花时间"——按**对我个人的价值**排序，而不是按全网热度。
  - 拥有一份永久、可搜索的个人记忆，刷过的内容不再遗忘、可回溯利用。
  - 一眼看出"今天 AI 圈在聊什么"（话题趋势）。
- **每屏的主任务**：扫排序后的 feed → 用 👍/👎 校准排序 → 关键词 + 语义搜索过往条目 → 查看话题趋势。

## Product Purpose

把四个 AI 资讯源聚合起来，去重（URL 规范化 + 内容 hash + 语义近似）、用便宜信号预筛后调 LLM 批量打分、按合成分排序、产出一句话摘要、聚类成话题并绘制趋势曲线，所有条目永久入库、可搜索、可收藏。

存在的理由：公共 AI 日报（TLDR AI、Ben's Bites、smol.ai AINews）给所有人同一份"大锅饭"；本工具只为一个人排序、并替这个人记住一切。护城河在**排序质量 + 个人记忆**，不在覆盖更多源。

成功的样子：

- 每天 feed 的 top ~20 里有足够多条是我之前不知道、且觉得有价值的（用反馈数据量化）。
- 不再需要手动刷四个平台。
- 能搜到"上周关于 agent 的讨论"（关键词 + 语义）。
- 趋势页能准确反映当天 AI 热议话题。
- LLM 打分成本每日可控（预筛把调用量压到候选集）。

## Brand Personality

- **三个词**：安静、专注、克制（quiet / focused / restrained）。
- **声音与语气**：冷静、精准、低噪音。界面以中文为主，同时展示英文摘要；界面应该往后退，让内容自己说话。
- **情绪目标**：让人确信"排在最上面的就是值得我花时间的"，是一种克制的笃定，而不是原始信息流那种焦虑的水龙头感。
- **视觉**：Awesomic 浅色系统——Mist 瓷白画布 + 白卡，UI 层几乎无彩色：唯一的填充动作色是近黑 `#09090b`（白字），ember 橙（`#ff5a00` / 文字用 `#c2410c`）只作极少量、徽标尺寸的点缀（eyebrow、收藏星、运行脉冲、accepted 标签），详见 DESIGN.md。这是服务于产品的视觉系统，不是营销品牌标识。

## Anti-references

- 不要像信息过载、广告塞满的内容流 / 无限滚动的水龙头。
- 不要像到处堆砌千篇一律卡片网格的通用 SaaS 仪表盘。
- 不要像严肃呆板的企业后台工具。
- 不要沦为 PRD 里点名的核心失败模式——"又一个浪费时间的信息流"。
- 不要做成面向所有人、千人一面的公共 AI 日报。

## Design Principles

1. **排序即产品。** 信号优先于覆盖。价值在于"什么值得我花时间"，不在 feed 是否齐全；要把排序理由（分数 / reason）露出来，让我信任这个顺序。
2. **界面后退，内容发声。** 默认安静，强调色只服务于动作，不让任何装饰跟我要读的内容抢注意力。
3. **有价值的东西一概不丢。** 一切永久留存、可搜索（关键词 + 语义），收藏永不过期。个人记忆是头等功能，不是事后补丁。
4. **反馈回路看得见、有后果。** 👍/👎 必须明显地改变排序；校准 feed 是"使用"的一部分，而不是埋在设置里的开关。
5. **为一个人而造。** 按本人口味和每日习惯优化，胜过追求通用；有主见的默认值好过可配置的平庸。

## Accessibility & Inclusion

单人自用工具，没有正式的 WCAG 合规强制要求。基线：

- 正文文本满足 WCAG AA 对比度（≥4.5:1），让克制的配色始终清晰可读；占位符同样要够清晰。
- 尊重 `prefers-reduced-motion`，为动效提供降级（淡入或瞬时切换）。
- 内容双语（中文标题/摘要 + 英文源摘要），两者都要保持易读。
