## 反模式判定

- **主观（LLM）**：不是 AI slop。无卡片网格/gradient text/hero-metric/side-stripe；玻璃笔记面板、mono 标签、稀缺色都能追溯到 DESIGN.md 仪器语言。两处模板惯性：① 一屏九个同形态 mono 小节标签（概述/核心要点/关键数据/为什么值得记/术语/全文/译文/原文/讨论），层级互不区分；② 推文条目硬套文章阅读器模板，同一句话出现四遍。
- **确定性扫描（CLI）**：7 个详情页 markup 文件，退出码 0，零发现。
- **页内 overlay（浏览器）**：14 条命中，但采集时页面处于"正在整理"占位态（恰因评审 A 实测触发了取消收藏删笔记）。有效命中：skipped-heading（h1→h3 跳级，与评审 A 的"读屏大纲残缺"互相印证）、text-overflow（sidebar__user-mail 溢出 45px，全局 shell 的真实 bug，评审 A 未发现）、tiny-text 10px（侧栏邮箱）。误报：dark-glow / ai-color-palette / thin-border-wide-shadow 是 DESIGN.md 锁定的 nebula glow 设计语言；两条 overused-font 自相矛盾（33% vs 50%，均未过半）；nested-cards ×2 命中侧栏 status 卡和占位态 notice，属仪器语言刻意设计。line-length ×2（139/149 字符摘要行）作用于占位态 DOM，正常态待复核。

## 认知负荷（8 项 checklist）

失败 2 项，勉强 1 项（中等负荷）：
- ❌ 分块 ≤4：笔记"关键数据·结论"实测 13 条 bullets、核心要点 6 条，LLM 输出无上限直接渲染
- ❌ 渐进披露：默认"对照"把双语全文一次性摊开（长文移动端整页 22,853px）；35 条讨论全量渲染无折叠
- ⚠️ 工作记忆：长文中要记住"切换器在区头"才能换语言

## 情感旅程（峰终）

- **到达（峰）**："← 信号流" + 标题 + 概述 3 秒内回答"值不值得读"，全页最好的时刻。
- **阅读中（谷）**：推文条目同一句话重复四遍，"全文"尽头是截断的 RT，落空。
- **高风险时刻（最深谷）**：取消收藏 = 静默销毁笔记，唯一不可逆动作恰好零防护。
- **离开（终，好）**：history.back 保留 feed 滚动与筛选，peak-end 的 end 做对了。

## 做得好的地方

1. **返回路径被当真了**：BackLink 优先 router.back() 保留 feed 滚动/筛选，直达才 push fallback；from=feed 让返回标签正确切换。阅读工具最高频动作被认真对待。
2. **双语架构工程与设计合一**：两个语言 pane 服务端渲染，客户端只有切换 state；对照列防长 token 溢出，<760px 自动堆叠；原文 muted 后退、译文在前，主次明确。
3. **讨论区把 markdown 墙解析成真实评论树**：monogram、分数 chip、回复缩进线，且只有唯一榜首评论的火焰分数点琥珀——稀缺色纪律执行到了逻辑层。

## 优先问题

**[P1] 取消收藏静默销毁 KB 笔记，无确认、无 undo**
- What：已收藏态点 .kb-save 一次 → PATCH isFavorited:false → 服务端立即 DELETE kb_entries（笔记、译文、评论、R2 图片全删）。aria-label 只说"点击移除"。评审实测一次 on→off 就把 1520 打回"正在整理"。
- Why：全页唯一不可逆动作放在页头右上最顺手的位置，误触成本是等 LLM 重建（时间+钱）。与产品设计原则第 3 条"有价值的东西一概不丢"直接冲突。
- Fix：取消收藏与删除笔记解耦（软删除/宽限期），或至少已收藏态点击给一次轻量确认；failed 文案别再推荐"取消再重收藏"这条破坏性重试路径。
- 建议命令：/impeccable harden

**[P2] 语言切换器 role="tablist" 是假 tabs**
- What：kb-seg 用 tablist/tab + aria-selected，但三个 tab 全 tabIndex=0、方向键实测无效、无 aria-controls/tabpanel。detector 同时抓到 h1→h3 跳级，读屏标题导航到不了正文和讨论。
- Why：读屏宣告"选项卡"但 ARIA 惯例操作失败；键盘用户多按两次 Tab。
- Fix：补全 tabs 模式（roving tabindex + 方向键）或降级为普通按钮组；"全文""讨论"从 span 改为真实 h2。
- 建议命令：/impeccable harden

**[P2] 移动端默认"对照"+ 切换器不跟随**
- What：BilingualBody 默认 compare，移动端堆叠后整页 22,853px；切换器只在"全文"区头，滚深后不可达。
- Why：手机上没人同时读两遍全文；想切回"译文"得往回滚十几屏。
- Fix：≤760px 默认 zh 或记住上次选择；或让 kb-read__head 在长文内 sticky。
- 建议命令：/impeccable adapt

**[P2] 推文条目套文章模板，"全文"名不副实**
- What：twitter 源 bodyMd 是截断的 ingest 文本，却放进"全文+译文/原文/对照"完整阅读器；概述/英摘/译文/原文四次重复同一句话。
- Why：标签承诺"全文"却交付截断文本；重复内容稀释"界面后退，内容发声"。
- Fix：twitter/短内容走引用卡片形态（正文一次 + 显著"读原推"出口），或至少改标签并提示截断。
- 建议命令：/impeccable shape

**[P3] 笔记 bullets 无上限 + violet 语义稀释**
- What：13 条 bullets 直接渲染；violet 用在九个不可点小节标签上，与 violet 链接同屏。
- Why：违反分块原则，笔记从提炼退化成搬运；不可点元素用链接色会训练用户忽略真链接。
- Fix：facts 设上限（6 条 + 折叠）；小节标签改 --text-faint/--smoke，violet 还给可点击物。
- 建议命令：/impeccable polish
